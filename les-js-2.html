Для чего JavaScript нужен в Web-разработке?

Для создания динамических страниц

До это момента мы делали верстку с помощью статических страниц, т.е. которые практически не реагировали на действия пользователя.

Но некоторые приёмы для визуальный отклика мы уже использовали с помощью CSS, например :hover :active @keyframes

Ещё раз повторим, чтобы начать использовать JavaScript, нужно ответить на 3 вопроса:

  • Что такое алгоритм?
  • Что такое язык программирования?
  • Что такое программа?

Алгоритм - это последовательность действий для достижения цели.

Язык программирования - это язык для общения с машинами, в нашем случае - с браузером. Язык программирования состоит из набора команд. Каждая команда имеет только одно значение.

Программа - это алгоритм, записанный на языке программирования. На JavaScript мы будем писать именно такие программы, заставляя браузер следовать определённому алгоритму.

Свойства и методы (действия)

Очень часто внутри круглых скобок что-то написано - это что-то называется аргументамифункции. В данном случае аргумент функции это строка "Hello, World!"

  • Напомните мне почему "Hello, World!" это строка?
  • Как это определяется в языках программирования?

Метод или функция log() объекта консоль, умеет делать, только одно действие, печатать в консоль переданный аргумент. Обратите внимание аргументом можно передать ни только текст, но и число например.

Зачем нам нужна консоль или писать что-то в консоль?

  • Чаще всего для вывода сообщений об ошибках или предупреждениях,
  • показать полезную информацию для разработчика или тестировщика,
  • так же ее можно использовать как песочницу, чтобы поиграться и быстро проверить какой-то код,
  • или использовать как калькулятор.

Запускаем VS Code

Создадим файл index.html и напишем в нём стандартную верстку через “!” 

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>JavaScript - 1 Занятие</title>
</head>

<body>

  <script>
    // Это однострочный комментарий
    // Выводим в консоль сообщение "Привет, мир!"
    console.log('Привет, мир!')
  </script>
</body>

</html>

Комментарии нужны для помощи программисту, они помогают понять, что делает код рядом с которым они написаны. Используются как заметка на будущее, чтобы не забыть что было сделано или что нужно сделать.

Я привык писать в комментариях теорию к теме которую изучаю, потом удобно просматривать и вспоминать изученные темы.

Запускаем Live Server, переходим в консоль и проверяем результат (он будет точно такой как и в прошлом примере через инструменты разработчика)

Чтобы запустить сайт через Live Server нужно:

  • В правом нижнем углу нажать на кнопку Go Live и откроется браузер по умолчанию с нашей страницей index.html.
  • В нужном HTML файле нажать ПКМ и выбрать пункт меню Open With Live Server

То как мы написали JS код внутри HTML файла является рабочим вариантом, но считается плохой практикой, когда мы всё пишем в одном файле, тут и верстка (дизайн) тут и программный код (логика) … смешались кони люди …

Нужно код на JS вынести в отдельный файл (как мы делали с файлами стилей css), назовём его script.js и не забудем подключить его в нашей верстке.

В файле script.js пишем код на JavaScript (теги <script> писать не нужно) 

// Это однострочный комментарий
// Выводим в консоль сообщение "Привет, мир!"
console.log('Привет, мир!')

В файле index.html подключаем внешний файл script.js так же в самом низу страницы перед тегом </body> 

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>JavaScript - 1 Занятие</title>
</head>

<body>

  <!-- Подключаем внешний файл script.js -->
  <script src="script.js"></script>
</body>

</html>

Команда alert()

Команда alert() ― ещё один способ посмотреть на данные. Работает точно также, как и console.log(), но результат оказывается не в консоли, а во всплывающем окне браузера. 

console.log('Привет, мир!')

// Используем встроенную функцию alert()
alert('Изучать JavaScript - это офигенно, отвечаю!')

На что инструкции похожи из жизни?

Например есть инструкция как что-то приготовить (рецепт), как мы будем готовить бутерброд?

Проведем очень наглядную демонстрацию 👀 Назовем ее бутербродный алгоритм.

Нужно два человека, которые будут исполнителями кода, по умному интерпретаторами. Они будут интерпретировать наши действия, то что программист им скажет делать.

Все остальные в аудитории будут программистами. Результат будет зависеть от такого насколько подробный и правильный алгоритм мы будем задавать нашим интерпретаторам.

Арифметические операции

В JavaScript можно производить арифметические операции с числами: складывать, умножать, делить, вычитать, возводить в степень, вычислять остаток от деления. Всё по правилам математики.

Операторы

Перед тем, как двигаться дальше, разберём базовую терминологию. Знак операции, такой как ”+” называют оператором. Операторы выполняют операции над определенными значениями, которые называются операндами.

Linter

Когда мы ошибаемся в написании синтаксиса, у нас есть прекрасный помощник, который называется Линтер, обычно он уже есть в VSCode, но можно установить и  настроить его различный версии.

На что это похоже?

Допустим когда мы набираем текст в офисной программе (например Word) и делаем орфографическую ошибку, она у нас подчеркивается красным цветом.

Точно также работает, Линтер в программном коде, он будет подчёркивать красным там где у нас ошибка и выдавать подсказку как ее исправить.
